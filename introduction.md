# Introduction to Pockgin

Pockgin is a **static-first plugin registry** for [PocketMine-MP](https://github.com/pmmp/PocketMine-MP).

## Philosophy

- **No backend runtime** - everything is static JSON + HTML
- **No database** - data lives in Git and is generated by CI
- **Zero cost** for the core flow - runs on GitHub Pages or Cloudflare Pages
- **Git-based workflow** - PRs are the moderation layer
- **All open source** - every repo in the Pockgin ecosystem is public

## How It Works

1. **Developers** maintain plugins in their own repos and use the standard Pockgin build workflow.
2. **Moderators** review submissions and approve specific release tags.
3. **CI pipelines** sync data from GitHub, generate static JSON, and deploy the website.
4. **Users** browse the website, view plugin details, and download `.phar` files.

## Key Principle

**Merge PR = Publish approved tag.**

A plugin only appears on the public website after its registry entry has been merged into the main branch.

## Repositories

| Repo | Purpose |
|------|---------|
| `pockgin/pockgin` | Main website + registry data |
| `pockgin/cli` | Standardized build CLI tool |
| `pockgin/plugin-template` | Template repo for new plugins |
| `pockgin/scripts` | Data pipeline (validate, sync, generate) |
| `pockgin/docs` | This documentation |
| `pockgin-archive` (org) | Archived forks/mirrors of approved plugins |
